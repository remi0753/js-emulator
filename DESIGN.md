# jscpu-os 設計仕様書

JavaScript で実装する仮想レジスタマシン CPU と、その上で動く
JavaScript 製 OS の設計ドキュメント。

最初のゴール: **プリエンプティブ・マルチタスク**
(タイマ割り込み相当で複数プロセスを時分割実行する)。

---

## 0. レイヤー構成

```
┌────────────────────────────────────────┐
│ ユーザープログラム (バイトコード)          │  アセンブラで生成
├────────────────────────────────────────┤
│ OS 層 (JavaScript)                       │  PCB / スケジューラ / syscall / loader
├────────────────────────────────────────┤
│ 仮想ハードウェア CPU (JavaScript)          │  fetch-decode-execute / メモリ / 割り込み
└────────────────────────────────────────┘
```

設計の中心は **CPU と OS の境界 (§6)**。
CPU は「一定サイクル実行したら必ず JS 側へ制御を返す」よう作る。
これにより OS(JS)が状態を入れ替え、コンテキストスイッチを実現する。

---

## 1. レジスタ

すべて 32bit 符号なし整数として保持(演算時に符号付き解釈する命令あり)。

| レジスタ | 用途 |
|----------|------|
| `R0`–`R7` | 汎用レジスタ(8 本) |
| `PC` | プログラムカウンタ(次に実行する命令のアドレス) |
| `SP` | スタックポインタ(空き先頭を指す。push で減算する下方成長スタック) |
| `FLAGS` | 状態フラグ(下記) |

### FLAGS のビット

| ビット | 名前 | 意味 |
|--------|------|------|
| 0 | `ZF` | 直前の演算結果が 0 |
| 1 | `SF` | 結果の最上位ビット(符号)が 1 |
| 2 | `CF` | キャリー/ボロー発生 |
| 3 | `IF` | 割り込み許可(`EI`/`DI` で操作) |

条件分岐は `CMP a,b`(= `a - b` を捨てる)後のフラグで判定する。

---

## 2. メモリモデル

- メモリは `Uint8Array`。既定サイズ 64 KiB(アドレス `0x0000`–`0xFFFF`)。
- ワード = 32bit、**リトルエンディアン**。
- スタックは高位アドレスから下方に成長(`PUSH` で `SP -= 4`)。

### プロセス分離の方針(重要)

- **v1**: プロセスごとに独立したメモリイメージ(別 `Uint8Array`)を持つ。
  コンテキストスイッチでレジスタとメモリ参照の両方を差し替える。
  → MMU 無しで完全分離が手に入る(実装が単純)。
- **v2(将来)**: 物理メモリ 1 枚 + `BASE`/`LIMIT` レジスタによる
  アドレス変換と保護違反フォルトを追加(本物の MMU に近づける)。

---

## 3. 命令エンコーディング

可変長。先頭 1 バイトがオペコード、続いてオペランドがオペコード仕様の順で並ぶ。

オペランドの種類:

| 種別 | サイズ | 内容 |
|------|--------|------|
| `reg` | 1 byte | レジスタ番号(0–7) |
| `imm` | 4 byte | 即値(リトルエンディアン 32bit) |
| `addr` | 4 byte | 絶対メモリアドレス(ラベルはアセンブラが解決) |

デコーダはオペコードから「オペランドの並び」を仕様テーブル(§5)で引いて読む。
アセンブラとデコーダは同じテーブルを共有する。

---

## 4. 命令セット

### データ移動
| ニーモニック | オペランド | 動作 |
|--------------|-----------|------|
| `NOP` | — | 何もしない |
| `MOV` | `rd, imm` | `rd = imm` |
| `MOVR` | `rd, rs` | `rd = rs` |
| `LOAD` | `rd, addr` | `rd = mem32[addr]` |
| `STORE` | `rs, addr` | `mem32[addr] = rs` |
| `LOADR` | `rd, rs` | `rd = mem32[rs]`(レジスタ間接) |
| `STORER` | `rd, rs` | `mem32[rd] = rs`(レジスタ間接) |

### 算術・論理(結果に応じて ZF/SF/CF を更新)
| ニーモニック | オペランド | 動作 |
|--------------|-----------|------|
| `ADD` | `rd, rs` | `rd += rs` |
| `SUB` | `rd, rs` | `rd -= rs` |
| `MUL` | `rd, rs` | `rd *= rs` |
| `DIV` | `rd, rs` | `rd = floor(rd / rs)`(rs=0 はフォルト) |
| `MOD` | `rd, rs` | `rd = rd % rs` |
| `AND` `OR` `XOR` | `rd, rs` | ビット演算 |
| `NOT` | `rd` | `rd = ~rd` |
| `SHL` `SHR` | `rd, rs` | シフト |
| `INC` `DEC` | `rd` | `rd ± 1` |
| `CMP` | `rd, rs` | `rd - rs` を捨ててフラグだけ更新 |

### 制御フロー
| ニーモニック | オペランド | 条件 |
|--------------|-----------|------|
| `JMP` | `addr` | 無条件 |
| `JZ` / `JNZ` | `addr` | `ZF==1` / `ZF==0` |
| `JG` `JGE` `JL` `JLE` | `addr` | 符号付き比較(SF/ZF から判定) |
| `CALL` | `addr` | `PC` を push してジャンプ |
| `RET` | — | スタックから `PC` を pop |

### スタック
| ニーモニック | オペランド | 動作 |
|--------------|-----------|------|
| `PUSH` | `rs` | `SP -= 4; mem32[SP] = rs` |
| `POP` | `rd` | `rd = mem32[SP]; SP += 4` |

### システム
| ニーモニック | オペランド | 動作 |
|--------------|-----------|------|
| `INT` | `imm` | ソフト割り込み。CPU を停止し OS へ制御を返す(§6) |
| `EI` / `DI` | — | `IF` を 1 / 0 にする |
| `HLT` | — | CPU 停止(プロセス終了 or アイドル) |

> タイマ割り込みは命令ではなく、CPU の「クォンタム切れ」で表現する(§6)。
> よって本物の IDT/IRET は v1 では持たず、OS が JS 側で制御する。

## 5. オペコード仕様テーブル(抜粋・確定は実装時)

CPU とアセンブラが共有する単一の真実。形式は
`MNEMONIC: { opcode, args: [...] }`。

```js
const ISA = {
  NOP:   { opcode: 0x00, args: [] },
  MOV:   { opcode: 0x01, args: ['reg', 'imm'] },
  MOVR:  { opcode: 0x02, args: ['reg', 'reg'] },
  LOAD:  { opcode: 0x03, args: ['reg', 'addr'] },
  STORE: { opcode: 0x04, args: ['reg', 'addr'] },
  // ... ADD/SUB/.../JMP/CALL/INT/HLT を同形式で続ける
  INT:   { opcode: 0x40, args: ['imm'] },
  HLT:   { opcode: 0xFF, args: [] },
};
```

## 6. CPU と OS の境界(最重要)

### CPU の実行 API

```js
cpu.run(maxCycles) // -> { reason, int? }
```

`maxCycles` 命令を実行する間に、以下が起きたら即座に JS へ戻る:

| `reason` | 発生条件 | OS の対応 |
|----------|----------|-----------|
| `'quantum'` | `maxCycles` 到達 | 状態を保存し次プロセスへ → **プリエンプション** |
| `'int'` | `INT n` 実行(`result.int = n`) | syscall 処理。必要なら再スケジュール |
| `'halt'` | `HLT` 実行 | プロセス終了として扱う |
| `'fault'` | 不正オペコード / 0 除算 / 範囲外 | プロセス強制終了 |

### コンテキスト(プロセスの実行状態)

```js
{ regs: [R0..R7], pc, sp, flags, mem /* v1: 専用メモリイメージ */ }
```

OS はこれを **PCB(Process Control Block)** に保持し、
`cpu.loadContext(pcb)` → `cpu.run(QUANTUM)` → `cpu.saveContext(pcb)` を回す。

### プリエンプティブ・マルチタスクの実現

```
OS のメインループ(疑似コード):

  while (readyQueue が空でない) {
    pcb = readyQueue.shift()        // ラウンドロビン
    cpu.loadContext(pcb)
    r = cpu.run(QUANTUM)            // 例: QUANTUM = 1000 命令
    cpu.saveContext(pcb)

    switch (r.reason) {
      'quantum': readyQueue.push(pcb); break          // 時間切れ → 末尾へ
      'int':     handleSyscall(pcb, r.int)            // syscall
                 if (pcb.alive) readyQueue.push(pcb); break
      'halt':    終了処理(pcb); break                 // キューに戻さない
      'fault':   エラー出力して終了処理(pcb); break
    }
  }
```

ここで `QUANTUM` 命令ごとに必ず制御が戻ることが、
ハードウェアタイマ割り込みの代わりとなり、プリエンプションを成立させる。

## 7. システムコール規約

| レジスタ | 役割 |
|----------|------|
| `R0` | syscall 番号(入口)/ 戻り値(出口) |
| `R1`,`R2`,`R3` | 引数 |

`INT 0x80` で OS の `handleSyscall` を呼ぶ(Linux 風)。

### 初期 syscall

| 番号 | 名前 | 引数 | 動作 |
|------|------|------|------|
| 0 | `EXIT` | R1=終了コード | プロセス終了 |
| 1 | `WRITE` | R1=文字コード | コンソールへ 1 文字出力 |
| 2 | `YIELD` | — | 自発的に CPU を手放す(末尾へ) |
| 3 | `GETPID` | — | R0 に PID |
| 4 | `SPAWN` | R1=プログラムID | 新規プロセス生成、R0 に子 PID |
| 5 | `SLEEP` | R1=ティック数 | 指定時間スリープ(ブロック) |

## 8. デバイス(段階的に追加)

- **コンソール**: v1 は `WRITE` syscall で `process.stdout` に出力。
- **タイマ**: v1 はクォンタムで代替。v2 で実割り込みベクタ化。
- **キーボード入力**: v2 でブロッキング `READ` syscall を追加。

## 9. 実装ロードマップ

1. `src/cpu.js` — メモリ + CPU + ISA テーブル + `run/loadContext/saveContext`
2. `src/assembler.js` — ニーモニック→バイトコード(ラベル解決つき)
3. 単体テスト: 1 プロセスで算術・分岐・`WRITE`・`HLT` を確認
4. `src/os.js` — PCB / ラウンドロビンスケジューラ / `handleSyscall` / loader
5. デモ: 2 プロセスを交互に動かし、出力がインターリーブすることを確認
   (= プリエンプティブ・マルチタスクの成立)
6. 拡張: `SPAWN`/`SLEEP`、その後 v2(MMU、実割り込み、shell/FS)

---

## 10. 設計上の決定事項(まとめ)

- レジスタマシン(R0–R7 + PC/SP/FLAGS)、32bit、リトルエンディアン。
- 命令は可変長、ISA テーブルを CPU/アセンブラで共有。
- **CPU は `run(maxCycles)` で必ず JS へ戻る** → OS が JS で時分割制御。
- タイマ割り込み = クォンタム切れ。syscall = `INT 0x80`。
- v1 はプロセスごとに独立メモリで分離。MMU と実割り込みは v2。
